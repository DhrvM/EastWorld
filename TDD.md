# Technical Design Document: SynthSpace API

**Goal:** A WebAPI to provision isolated synthetic environments populated by persona-driven AI agents (Synths) with customized tool access, memory, and networking rules, designed for product testing and market simulation.

---

## 1. Relational Database Schema (Supabase / PostgreSQL)

To maintain the state of the environments and synths across API calls, you need a strict relational schema.

### Table: `environments`

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `uuid` (PK) | Unique identifier for the simulation. |
| `e2b_sandbox_id` | `string` | The active remote Sandbox ID generated by E2B. |
| `status` | `enum` | `INITIALIZING`, `RUNNING`, `PAUSED`, `TERMINATED`. |
| `max_turns` | `integer` | Hard limit on how many messages can be exchanged. |
| `current_turn` | `integer` | Counter tracking the simulation time. |
| `created_at` | `timestamp` | Environment spin-up time. |

### Table: `synths`

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `uuid` (PK) | Internal DB identifier. |
| `env_id` | `uuid` (FK) | Reference to the `environments` table. |
| `synth_name` | `string` | Human-readable name (e.g., "Alex"). |
| `supermemory_user_id` | `string` | Unique tag used in Supermemory to track long-term state. |
| `persona_prompt` | `text` | The core system instructions defining behavior. |
| `allowed_tools` | `jsonb` | Array of tool names (e.g., `["execute_python", "read_file"]`). |
| `allowed_connections` | `jsonb` | Array of `synth_id`s this agent is permitted to talk to. |

### Table: `event_logs` (The Transcript)

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `uuid` (PK) | Log entry ID. |
| `env_id` | `uuid` (FK) | The environment this event happened in. |
| `actor_id` | `string` | `synth_id`, `system`, or `human`. |
| `event_type` | `enum` | `MESSAGE`, `TOOL_CALL`, `TOOL_RESULT`, `SYSTEM_ALERT`. |
| `payload` | `jsonb` | The actual message content or standard output from a tool. |

## 2. Tools & Secure Execution (The E2B Bridge)

Tools are implemented via LLM Function Calling. When an LLM outputs a tool call, the backend catches it and routes it to the secure E2B Sandbox.

### 2.1 Tool Schemas (Provided to the LLM)

When prompting the LLM, you inject the tools based on the Synth's `allowed_tools` array.

**Example: The `execute_python` Tool Schema**

```json
{
  "type": "function",
  "function": {
    "name": "execute_python",
    "description": "Execute python code in a Jupyter notebook cell within your secure sandbox. Use this to analyze data, make API requests, or process files.",
    "parameters": {
      "type": "object",
      "properties": {
        "code": {
          "type": "string",
          "description": "The raw Python code string to execute."
        }
      },
      "required": ["code"]
    }
  }
}
```

### 2.2 The Execution Bridge (Python Implementation)

When the LLM decides to use `execute_python`, your backend runs this exact SDK logic to securely execute it in the cloud.

```python
from e2b_code_interpreter import Sandbox

def execute_tool_in_sandbox(e2b_sandbox_id: str, tool_name: str, arguments: dict):
    # Connect to the long-running environment container
    sandbox = Sandbox.connect(e2b_sandbox_id)

    if tool_name == "execute_python":
        code = arguments.get("code")

        # Execute securely inside the E2B microVM's Jupyter server
        execution = sandbox.notebook.exec_cell(code)

        if execution.error:
            return f"Error: {execution.error.name} - {execution.error.value}"

        # Return standard output and any display data (like charts)
        return {
            "stdout": execution.logs.stdout,
            "stderr": execution.logs.stderr,
            "results": [res.text for res in execution.results]
        }

    elif tool_name == "read_file":
        path = arguments.get("path")
        content = sandbox.files.read(path)
        return content
```

## 3. The Cognitive Loop (Supermemory + LLM)

Before a Synth generates a response, it must fetch its long-term context (Memories + Static Profile) and inject it into the prompt.

### 3.1 Fetching Context (Supermemory API)

We use the Supermemory Python SDK. By passing the `supermemory_user_id` as the `container_tag`, the agent retrieves only memories relevant to its specific persona and the current situation.

```python
from supermemory import Supermemory

sm_client = Supermemory(api_key="...")

def get_synth_context(supermemory_user_id: str, current_situation: str):
    # Supermemory automatically searches its vector-graph database
    profile_data = sm_client.profile(
        container_tag=supermemory_user_id,
        q=current_situation  # What the agent is currently looking at
    )

    static_facts = "\n".join(profile_data.profile.static)
    dynamic_context = "\n".join(profile_data.profile.dynamic)
    relevant_memories = "\n".join(
        r.get("memory", "") for r in profile_data.search_results.results
    )

    return f"""
    [YOUR BACKGROUND]:
    {static_facts}

    [YOUR CURRENT STATE]:
    {dynamic_context}

    [RELEVANT MEMORIES]:
    {relevant_memories}
    """
```

### 3.2 The Synth's Turn (The Event Loop)

When it is a Synth's turn to act in the environment, the backend executes this flow:

1. **Context Assembly:**
   - Fetch `persona_prompt` from Supabase.
   - Call `get_synth_context()` from Supermemory.
   - Fetch the last N messages from `event_logs` where `env_id = current_env`.

2. **LLM Invocation:**
   - Pass the assembled context and the allowed JSON Tool Schemas to LiteLLM / OpenAI.

3. **Action Routing:**
   - *If LLM returns a standard text message:* Verify the target Synth is in `allowed_connections`. If yes, broadcast the message to the Environment and write to `event_logs`.
   - *If LLM returns a Tool Call:* Route the JSON payload to `execute_tool_in_sandbox()`. Take the execution result, append it to the chat history as a "Tool Response", and feed it *back* to the LLM so it can interpret the results.

4. **Memory Ingestion:**
   - Asynchronously send the final action/message back to Supermemory so the Synth remembers doing it in future turns.
